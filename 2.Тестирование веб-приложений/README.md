# **Задачи проекта**

1. Составить тестовую документацию, выполнить проверки, завести баг-репорты.

2. Протестировать новый вид транспорта по готовому чек-листу. Необходимо в Charles: подменить ответы от бэкенда и настроить автоматические ответы. 

# **Ход проекта**

1. Изучены макеты в Figma, составлен чек-лист на вёрстку следующих блоков: форма бронирования; элементы на навигационной карте: это иконки автомобилей и действия с ними.

2. Составлена следующую тестовую документацию: 
* чек-лист на логику окон «Способ оплаты» и «Добавление карты»,
* тест-кейсы на кнопку «Забронировать» и функциональность бронирования.

3. Проверено приложение по составленной документации в двух окружениях. На обнаруженные ошибки созданы баг-репорты в YouTrack.

4. Чтобы аэротакси отобразилось в интерфейсе, перехвачен ответ бэкенда, который содержит типы транспорта, и подставлен новый тип — аэротакси.

(В Яндекс Маршруты планируют добавить аэротакси. Фронтенд уже готов, а бэкенд задерживается. В запросах фронтенда на типы транспорта нет аэротакси. Ответ тоже приходит без него.)
